# Sophia Forest 운영 워크플로우 (AI/IDE 공용)

## 목적
- `Sophia Forest`는 구현 자체가 아니라, 구현 상태/우선순위/문서 기준선을 관리하는 관제 레이어다.
- 작업은 코드에서 진행하고, 관제 기준은 Forest에서 관리한다.

## 기본 동선
1. 좌측 `소피아`(대분류) 클릭
2. 우측 `문서/계획 워크스페이스`에서 기준 문서 확인
3. 문서 상태를 `pending/review/confirmed`로 분류
4. 필요 시 `SonE 검토` 실행 후 상태 재판정
5. TODO(가중치) 등록 후 우선순위 순으로 구현 진행
6. 구현 반영 후 현황 동기화

## 문서 상태 규칙
- `pending`: 작성/업로드 완료, 아직 검토 전
- `review`: 검토 필요 또는 SonE 리스크/누락 존재
- `confirmed`: 기준선 확정, 구현 기준으로 사용 가능

## TODO 운영 규칙
- TODO는 `priority_weight`(0~100) 기준으로 정렬
- 권장 상태 흐름: `todo -> doing -> done`
- 기능 개발 시작 전 TODO를 먼저 등록하고 진행한다.

## 병렬 작업 규칙 (다른 AI/IDE 포함)
- 공통 기준: 루트 문서 상태 + TODO 상태
- 병렬 작업자는 문서 확정 여부를 먼저 확인하고 착수
- 충돌 가능 작업은 `review`로 전환 후 SonE 재검토

## 주의
- Forest는 “결정 근거와 상태”를 정리하는 곳이다.
- 소스 코드 수정 자체는 각 모듈/서비스에서 수행한다.
- 확정 문서는 읽기 기준선으로 사용하고, 직접 편집은 원본 파일에서 수행한다.
