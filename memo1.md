1. 핵심 질문 재정의

700GB / 700여 건 사건 “풀셋(교사원공소장판결문~재판자료)”을 콘텐츠 생산에 바로 쓰이는 형태로 재구조화하려면
	•	(A) 사건 단위로 검색·요약·인용이 가능해야 하고
	•	(B) 민감정보·비공개 자료가 섞인 상태에서 안전하게 가공·공유되어야 하며
	•	(C) 유튜브 제작 흐름(주제 선정→대본/쇼츠→근거자료 확보→검수)과 직결되어야 합니다.

즉 “폴더 정리 프로그램”이 아니라 사건 지식베이스 + 콘텐츠 생산 파이프라인으로 설계하는 게 손익이 맞습니다.

⸻

2. 구조 분석 (구조 / 효율 / 적용성 / 리스크)

2-1. 구조: “원본 보관층 / 가공층 / 공개층” 3계층이 필수
	1.	원본 보관층(증거보존)

	•	SSD에 그대로 “원본 읽기전용” 보관
	•	파일명/폴더 변경 최소화(감사·추적·분쟁 대비)

	2.	가공층(내부 작업용)

	•	사건 단위 인덱싱 + OCR/텍스트 추출 + 요약 + 태깅
	•	내부자(변호사/PD/실무자)만 접근

	3.	공개층(콘텐츠 제작/업로드용)

	•	개인정보 완전 마스킹된 “발췌본/근거캡처/요약본”
	•	영상 대본, 쇼츠 포인트자막, 썸네일 문구까지 연결

이 3층이 분리되지 않으면, 시간이 지나면서 원본 훼손 / 마스킹 누락 / 자료 유출이 반복됩니다.

2-2. 효율: “분류”는 폴더가 아니라 메타데이터 + 검색
700건이면 폴더 트리로는 한계가 빨리 옵니다.
핵심은 사건 카드(1건=1레코드) 를 만들고, 파일은 “첨부물”로 붙이는 구조입니다.
	•	사건 카드 필드(초기 최소셋)
	•	사건ID(내부), 사건명(가명), 사고유형, 피해결과(상해/사망), 당사자(가명)
	•	진행단계(수사/1심/2심/확정/합의), 결과(벌금/집유/실형/불기소/승소 등)
	•	핵심쟁점(예: 중앙선침범 해당 여부, 과실비율, 인과관계, 양형요소)
	•	콘텐츠 포인트(분노/반전/교훈/보험사 이슈/제도 이슈)
	•	공개가능등급(0~3)
	•	근거문서 링크(판결문, 공소장, 감정서 등)

이렇게 해야 “이번 주 쇼츠 10개 뽑기”가 검색 2번으로 끝납니다.

2-3. 적용성: 사무실 실제 운영을 생각하면 “권한/감사로그”가 먼저
	•	누가 어떤 문서를 열람/내보내기 했는지 기록(감사 로그)
	•	마스킹 완료된 자료만 외부 폴더로 export 가능
	•	“한 번에 업로드/공유” 같은 기능은 위험. 항상 공개층만 대상으로

2-4. 리스크: 가장 큰 지뢰 4개
	1.	개인정보/민감정보(주민번호, 주소, 연락처, 차량번호, 계좌, 병력, 진단서, CCTV 얼굴 등)
	2.	비공개 문서 범위(의뢰인 제공 자료, 내부 메모, 전략 문건, 상대방 제출자료 등)
	3.	저작권/제3자 권리(CCTV, 블랙박스, 사진, 감정서 일부)
	4.	사실관계 왜곡 리스크(요약·편집 과정에서 오해 발생)

따라서 프로그램에 “기능”보다 먼저 들어가야 하는 게
마스킹 파이프라인 + 공개등급 + 검수 워크플로우입니다.

3. 실행 전략 (우선순위 / 자원 / 리스크 포함)

3-1. 0단계: SSD 수령 직후 기본 원칙 확정
	•	원본은 읽기전용 보관(원본 폴더 잠금, 해시 생성)
	•	작업은 “가공층”에서만 복제본으로 수행

3-2. 1단계 MVP: “사건 카드 + 검색 + 공개층 export”만 먼저
MVP 목표는 단순합니다.
	•	사건 700건을 1) 찾고 2) 이해하고 3) 안전하게 뽑아내는 것

필수 기능 7개
	1.	사건 카드 생성(수동+일괄)
	2.	파일 자동 분류 후보 생성(판결문/공소장/교사원 등 문서 유형 추정)
	3.	텍스트 추출(OCR 포함) + 문서 내 키워드 인덱스
	4.	사건 태그/필터(사고유형/결과/쟁점/보험이슈/클립포인트)
	5.	“콘텐츠 아이디어 보드”(이번 달 주제 30개 자동 추천)
	6.	마스킹 체크리스트 + 공개등급 승인
	7.	공개층 export(영상팀 폴더로 안전 복사 + 워터마크/로그)

3-3. 2단계: 유튜브 제작 자동화 연결
여기서부터 “PD 파이프라인”이 붙습니다.
	•	사건 카드 → 영상 기획 템플릿 자동 생성
	•	판결문 핵심 문단 → “근거 캡처” 자동 생성(마스킹 후)
	•	쇼츠 포인트 자막 자동 생성(이미 가지고 있는 용PD 규칙 적용)  ￼
	•	쇼츠 자막 예시 포맷과 동일하게 출력  ￼

즉, 사건 하나를 선택하면
“기획안 + 근거팩 + 쇼츠 자막 초안”이 한 번에 나와야 생산성이 폭발합니다.

4. 사무실에 물어보고 확인받아야 할 방향성 체크리스트

여기서 답이 나오면 설계가 확정됩니다.

4-1. 법적/윤리 기준선(가장 먼저)
	•	콘텐츠로 공개 가능한 범위의 내부 기준이 있는지(없으면 이번에 정해야 함)
	•	의뢰인 동의가 필요한 케이스 기준
	•	판결문/공소장/기록 중 “절대 외부 반출 금지” 범주 정의
	•	영상에 쓸 때 “사실관계 표현 가이드”(단정 금지 표현, 익명화 수준)

4-2. 운영 워크플로우
	•	누가 사건을 등록하고 누가 검수 승인하는지(권한 분리)
	•	PD가 필요한 건 “이슈/감정포인트/판결요지/보험사 대응 포인트” 중 무엇인지 우선순위
	•	사건 1건당 목표 산출물:
	•	쇼츠 1개인지, 롱폼 1개인지, 시리즈인지

4-3. 기술/보안
	•	SSD는 사무실 내 어디에 보관, 누가 접근
	•	클라우드 사용 여부(원칙적으로 원본은 로컬, 공개층만 클라우드 권장)
	•	백업 전략(2중 백업, 랜섬웨어 대비 오프라인 백업)

4-4. 분류 체계 합의(초기 표준)
	•	사건 유형 표준(예: 보행자/자전거/이륜차/차대차/스쿨존/음주/뺑소니 등)
	•	결과 표준(형사/민사/행정 각각)
	•	쟁점 표준(과실, 인과관계, 중앙선, 신호위반, 안전의무, 양형요소 등)
	•	보험/공제조합/자배법/정부보장 등 “제도 태그” 표준
